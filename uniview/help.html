UniView

Updated Sat 6 Jun 2015 • tags uniview, apps

UniView is an HTML-based application for working with Unicode characters. You can look up or find characters (using graphics or fonts) and related information, view whole character blocks or custom ranges, select characters to paste into your document, paste in and discover unknown characters, search for characters using regular expressions, do hex/dec/ncr conversions, highlight character types, etc. etc. It supports Unicode 8.0 and is written with Web Standards to work on a variety of browsers.

This help file relates to Version 8.0.0 of UniView. Significant changes in this version include the addition of the new Unicode 8.0 codepoints, and fixes for the Show Age filter. For details of this and other changes see the change history.

The first three digits of the UniView version number reflect the version of Unicode that it supports. This version therefore supports Unicode version 8.0.0.

Quick overview

To get started, use the Show block selector, or add characters or code points to one of the fields on the Look up tab and hit the icon. UniView produces tables or lists of characters in the lower left of the window.

By default, clicking on a character in a list or table displays detailed information for that character to the right.

Use the toggle switch to add the characters to the Edit buffer box instead. Alternatively, you can paste characters directly into the Edit buffer box and hit to discover what they are.

UniView can do much more. Mouse over the first letter of a command name to reveal an orange link to context-sensitive help, or check out the rest of this user guide.

With UniView you can...

See a range of Unicode characters

Either select a Show block from the Show block selector, or type it using hex numbers into the Show range text box and click on .

The Show range field will accept various formats. The numbers must be in hexadecimal form and separated by a colon (the default), a hyphen, one or more spaces, or one or more periods. The numbers can be in the following formats: 1234, &#x1234;, \1234;, \u1234, U+1234. The actual number of hex digits can be between 1 and 6.

You can display the result as either a table of characters or a list (that includes names) by clicking the check box entitled Show range as list. If you use a list, you will also see the sub-block-level headings that are used in the Unicode Standard.

By default characters are shown as a table.

Unassigned character positions in a table are shown with a greyed out background (though you can change the colour, if you want).

View characters as graphics or font glyphs

Use graphics allows you to specify whether you want UniView to use font glyphs or graphics to display characters. Using font glyphs loads the page faster, but relies on you having a Unicode font or selection of fonts to cover all the needed Unicode code points. The default is to display character using graphics. Some of the graphics are downloaded from the decodeunicode server.

If you are want to find fonts, check out my script resource pages. When you select a Unicode block, a pointer to the relevant page appears at the top right of the page.

Search for one or more characters based on text in the Unicode database (eg. you know the name contains 'khwai')

Type in the string you want to search for in the box labelled Find and click on the Search button. (If it is already highlighted, you can hit enter or click on .)

You can use regular expressions in searches. For example, suppose you wanted to find all characters with the word 'tet'. You could type into the input field, \btet\b. (In UniView a colon can be used as a short form of \b so the example could have been written :tet:). The \b represents a word boundary. If you wanted to search for entries containing either the word 'tet' or the word 'tat' you can use the 'or' operator | as in \btet\b|\btat\b.

Another example: You want to search for 'alpha', but you only want results for the Latin characters (not the many Greek or mathematical results). Simply use the following search string latin.*alpha. The .* represents any number of intervening characters.

I haven't tested this feature to destruction, but most basic regular expressions that work in either JavaScript or PHP code should work.

Note that by default searches match against character names and alternative names in the main Unicode database, and also the information displayed for an individual character under the heading Description in the right panel (this information is taken from the NamesList.txt file on the Unicode site). You can limit the search using the Names, Descriptions and Other check boxes that appear which the Search button is highlighted. Other refers to alternative names.

You can also limit the search to the specific range of characters currently in a displayed list or table. To limit the search, select the check box labelled Local. Matching characters will be highlighted.

(Hint: You can then produce a list of just the highlighted characters by clicking on the icon next to Make list from highlights on the Filter tab. If you need to refine your search, you can then search again on this list, and so on.)

Discover what characters are in a string via cut and paste

Copy and paste the string into the box labelled Edit buffer and click on .

You can do a similar thing using the Look up tab. This can be useful if you want to investigate some characters without disturbing what is currently in the Edit buffer box. To do this, add the characters to the Find field on the Look up tab, and click on the Characters button below (or hit return or click on if it is already highlighted).

Find out about one or more characters, whose hex or decimal code point value you know

Type or paste the code point values in the box labelled Find, and hit the Hex or Dec button (or, if one of those is already highlighted, hit return or click on ). (See also the next point). By default, Find will find and list hex code points.

This field is very forgiving about the format of the text entered into the box. Most types of character escapes will be recognised, and you can even paste in surrounding text. For example, UniView will detect and list the characters referred to by code point in the text "the decomposition mapping is <U+CE20, U+11B8>, and not <U+110E, U+1173, U+11B8>." Of course, this is not foolproof, but should provide the desired results most of the time.

Link to the Unihan database for any han character

Add the character to the box labelled Edit buffer and click on . The Unihan information will be displayed in a separate page.

If you have more than one code point or character in the box, UniView will open a separate window containing Unihan information for each. If you try to open more than 5 windows, a warning will pop up.

You can also find a link to the Unihan database in the detailed character information in the lower right panel when a CJK Unified Ideograph is displayed there.

Convert to hex, decimal, NCR, percent-encoded escape formats, or view utf-8 or utf-16 code equivalents, etc.

Click on the icon next to the Edit buffer boxes to open the conversion tool. If there is a code point value or a string of characters in the box, values for those will be automatically shown when the conversion page opens.

(You can also look up a single character at a time from the detailed character information by clicking on the Conversion tool link.)

Set the height of the viewable area in the left panel

You can change the height of the display box in the lower left panel using Left panel max height (pixels) in the Options tab. Don't type 'px', just type the number.

This is particularly useful when you are dealing with lists on a small screen such as an iPad or netbook. If you set the height to something like 400px, you can scroll through a long list, but still see the details in the right panel when you click on a character in the list.

A new list of presets has been added below the above control. This may be expanded over time.

Highlight or list characters with a given General or Bidirectional property

Select a property from the Show properties drop down list on the Filter tab.

If the Local check box is selected this will only highlight characters in the currently displayed list or table.

Show when a character was added to Unicode

Click on the icon alongside Show age on the Filter tab. It shows version numbers only for characters added after Unicode version 1.1.

You can also find the same information on a character-by-character basis when details are displayed in the lower right panel.

To remove the information, click on alongside Show age.

Find information about a script

When you display a Unicode block in the lower left panel you will see the name of the block appear near the top right of the page, followed by the icon . Click on the icon to open a page of resource links in a separate window.

While working with characters in a list or table you can also...

Show detailed information for any character

Ensure that the large icon below the edit buffer is greyed out, ie. (if not, click or tap on it), then click on the character in the list or table. Details for that character are shown to the right of the table or list.

Quickly transfer all or one characters to the text area

Ensure that the large icon below the edit buffer is NOT greyed out, ie. it should look like this: (if not, click or tap on it), then click on the character in the list or table. The character will be appended to the Edit buffer box.

To copy all characters in the list or table into the Edit buffer box, click on the icon, which can be found just below the Edit buffer box.

Switch between viewing as a table or a list

Enable or disable Show range as list on the Look up tab.

This only applies if the lower left panel was populated by selecting a range using Show block or Show range, since these are the only functions that produce a table. (Lists show character names and hex code points, in addition to the character.)

Search the names or descriptions

Select the Local check box below the Find input field, then specify what you want to search for in the search input box. Search strings can be regular expressions, and you can specify what aspects of the information about a character are searched (see above for details). Characters that match will be highlighted.

Highlight the characters by property

Select the type of property you want from the Show properties selection list on the Filter tab, after ensuring that the adjacent Local check box is checked. Characters with the selected property will be highlighted. Properties available include general category and combining class or directionality.

Create a list of only those characters that are highlighted

On the Filter tab, click on next to Make list from highlights. The characters shown in the lower left panel will be reduced to a list of just those that were highlighted. (This is particularly useful for refining searches.)

Create a list of only those characters that are NOT highlighted

On the Filter tab, click on next to Make list from non-highlighted items. The characters shown in the lower left panel will be reduced to a list of just those that were not highlighted. (This is particularly useful for refining searches.)

Find out the decimal code point value for a character

Mouse over a character and the decimal code point value pops up in a tooltip. The decimal code point value is also shown in the right panel.

If mouseover doesn't work (for example on a mobile phone), display the details for the character and you will see the decimal code point listed.

Increase the size of the characters in a table

On the Options tab select a zoom factor from the pull down menu next to Left panel font size.

This only works when Use graphics is unchecked. You can, however, use the browser's zoom function instead, although that affects all text, rather than just the characters themselves.

Increasing or decreasing a browser's text zoom can multiply the effect of the selector.

Change the order or number of items on the lines in a list

This can be particularly useful when you want to copy and paste a list into another document. In the options, use the check boxes after List format to indicate what you want to see.

Switch between showing U+ before hex code point values

Toggle the check box labelled Show U+ in lists on the Options tab.

Switch between showing or hiding hex numbers around the table

Toggle the check box labelled Hide numbers around a table of characters on the Options tab.

While looking at the detailed character information you can also...

View the previous code point in the Unicode database

Click on the Previous button at the top of the detailed information pane.

View the next code point in the Unicode database

Click on the Next button at the top of the detailed information pane.

Display additional notes about characters where available

If there is a Character notes link and you click on it, a new page will open in a separate window. This page contains a list of all the characters in the block, and many of the characters have additional information. The notes are continuously growing. At the time of the release of UniView 7beta this is only available for Tibetan, but over coming days and weeks I will silently add more such links.

Look up information about that character in other databases

Click on the CLDR's Property demo link. A new window will open to show the entry for that character in the CLDR database. This provides additional, less commonly used data and properties relating to the character.

Click on the decodeUnicode link. A new window will open to show the entry for that character in the decodeUnicode database. decodeUnicode is a wiki where people can provide information about characters.

decodeUnicode.org is a wiki where people can contribute information about Show blocks and characters. It is developed at the Department of Design at the University of Applied Sciences in Mainz. The project is supported by the Federal Ministry of Education and Research (BMBF) and has the objectives of creating a basis for fundamental typographic research and facilitating a textual approach to the characters of the world for all computer users. (They also provide the graphic versions of characters for UniView.)

Click on the FileFormat link. A new window will open to show the entry for that character in the FileFormat database.

The FileFormat pages provide useful information for Java and .Net programmers.

Click on the Conversion tool link. A new window will open to show a number of possible alternative representations of the character, eg. numeric character entity references, percent escaped forms, hex and decimal code point information, etc.

Display the block to which the character belongs

Click on the link next to the subheading Show block and all characters in that block will be displayed as a list or table (according to your settings).

Working with the Edit buffer box

The Edit buffer box is associated with a set of controls for managing characters as text. It makes UniView like a character map or picker tool, but also much more.

To add characters to the box from a list or table, ensure that the large icon is not greyed out (if it is, click or tap on it), then click on the character in the list or table to the left. The character will be appended to the Edit buffer box.

Alternatively, you can paste text into this box, or edit it directly. The icon allows you to add spaces with a click.

The insertion point for characters echoed to the Edit buffer box can be changed in most browsers by just clicking where you want characters to appear (but not Internet Explorer, where characters are always added to the end of the line). You can also highlight a range of text and any typed or echoed characters will replace the highlighted range.

The icon is provided to simplify copying the text in the box. It highlights all the text in the box. (Particularly useful to check you have caught all combining characters.)

You can display a list of the characters in the box, with names and code points, by clicking on the icon. This is particularly useful for investigating text with characters you can't see or correctly identify. Simply paste the text into the text area and click on , and UniView will produce a list of the names and code points for all the characters in the box.

Conversely, clicking on the icon will copy to the Edit buffer box all the characters currently displayed in a list or table. This is particularly useful for capturing search results, or making a list of all characters in a block, etc.

You can convert the text to Unicode Normalization Form C or D (NFC or NFD) by clicking on either or . The change may not be obvious, but if you click on you should see any changes to the text listed below.

You can also convert characters to various escape or other forms by clicking on , or look up han characters in the UniHan database by clicking on . The icon clears all text from the text area.

When you click on any hyphen-separated lists in the text area box are expanded. For example, if you have A-Z in the text box, it will be replaced with ABCDEFGHIJKLMNOPQRSTUVWXYZ. You can expand multiple ranges at the same time. For example, A-Za-z0-9 will expand to include all intervening characters for the three ranges specified.

When you click on it simply reports the number of codepoints in the text area box.

Clicking on will display the characters in the Edit buffer box in the lower right part of the page with all relevant characters converted to uppercase, lowercase and titlecase. Characters that had no case conversion information are also listed.

Clicking on produces the same kind of output as clicking on the icon just above, but shows the mappings for those characters that have been changed, eg. e→E.

Controls explained

If you click on the first letter of the label of any of the controls listed below, you will open this document at the appropriate place for an explanation of that control.

Look up > Show block

Select a Unicode block from the pull-down list and the characters in the block will be displayed below. You can then click on characters to view detailed information about them or add them to the text area, etc.

You can display the result as either a table or a list (that includes names) by clicking the check box lower down entitled Show range as list.

Unassigned character positions are shown in the table with a greyed out background.

You can also specify a Unicode range by typing or pasting a hex code point range into the Show range box below this control.

Look up > Show range

If you type or paste a start and end code point value (in hex) into this control, the characters in the range will display below. Note that this can only be one contiguous range.

You can display the result as either a table or a list (that includes names) by clicking the check box lower down entitled Show range as list.

If the range you select does not fill a whole column when displayed as a table, surrounding characters are greyed out. (When displaying as a list, you will only see the characters in the range.)

The Show range field will accept various formats, making it easier to paste a range from elsewhere. The numbers must be in hexadecimal form but can be separated by either a colon (the default), a hyphen, one or more spaces, or one or more periods. The code point values themselves can be in the following formats: 1234, &#x1234;, \1234;, \u1234, U+1234. The actual number of hex digits can be between 1 and 6.

Look up > Find

This control accepts a list of codepoint values in hexadecimal or decimal, a list or characters or a search string. After adding content to the input box, click on the appropriate button below the control to activate lookup.(If the right button was already highlighted, you could just hit Enter or click on .

By default the control expects hex codepoint values. To look up a decimal value, click on the Dec button.

If the Characters button is selected, each character in the control will be processed. (Much of the time you will probably find it easier to use the Edit buffer box for this, but this control can be used if you want to look up some characters without disturbing a set of characters you are building up or using in the Edit buffer box.)

See the next section for more detail about using the Search function.

Click on to quickly clear the control.

This field is very forgiving about the format of the text entered into the box. Most types of character escapes will be recognised, and you can even paste in surrounding text. For example, UniView will detect and list the characters referred to by code point in the text "the decomposition mapping is <U+CE20, U+11B8>, and not <U+110E, U+1173, U+11B8>." Of course, this is not foolproof, but should provide the desired results most of the time.

Look up > Find > Search

This control allows you to search for text in the Unicode database, and returns a list of matching characters.

You can use regular expressions in searches. For example, suppose you wanted to find all characters with the word 'tet'. You could type into the input field, \btet\b. The \b represents a word boundary. If you wanted to search for entries containing either the word 'tet' or the word 'tat' you can use the 'or' operator | as in \btet\b|\btat\b. (In UniView a colon can be used as a short form of \b so the example could have been written :tet:).

Another example: You want to search for 'alpha', but you only want results for the Latin characters (not the many Greek or mathematical results). Simply use the following search string latin.*alpha. The .* represents any number of intervening characters.

I haven't tested regular expressions to destruction, but most basic regular expressions that work in both JavaScript and PHP code should work.

By default searches match against character names and alternative names in the main Unicode database, and also searches the information displayed for an individual character under the heading Description in the right panel. You can limit the search using the Names, Descriptions and Other check boxes that appear above the input box. Other refers to alternative names. If you want to set these before searching, just click on the Search button with nothing in the input box. It will bring up the checkboxes, which you can set as required. Then just enter your search string and click again on the Search button.

You can also limit the search to the characters currently in a list or table. To limit the search, select the check box labelled Local. Matching characters will be highlighted. (You can then produce a list of just the highlighted characters by clicking on the icon next to Filter > Make list from highlights. If you need to refine your search, you could then search again on this list, and so on.)

To clear the highlighting after doing a local search, just clear the Find input box, and hit the Search button again.

Filter > Show properties

This control allows you to search for characters with a particular property. It creates a list of matching characters.

By default, searches match against the characters in a list or table. Matching characters will be highlighted. (You can then produce a list of just the highlighted characters by clicking on the icon next to Filter > Make list from highlights. If you need to refine your search, you could then search again on this list, and so on.)

To enlarge the search to the whole of Unicode, deselect the check box labelled Local.

Filter > Show age

This control allows you to see when a character was added to Unicode (This currently only works when characters are displayed in a table.) It shows version numbers for characters added after Unicode version 1.1.

You can also find the same information on a character-by-character basis in the detailed information for that character (displayed bottom right in UniView).

To remove the information, click on the icon alongside, or display new data in the left panel.

Look up > Use graphics

If the check box is selected, all characters except those in the Edit buffer box and notes from the database will be shown as graphics, rather than text. The graphics are downloaded from the decodeunicode server.

If Unicode has recently issued a new version, it may take a while for the new characters to become available.

Look up > Show range as list

If the check box is selected, when you select a range using Show block or Show range the characters will be displayed as a list, rather than a table.

You can also use this to switch between table and list views of a range you have just selected.

Filter > Make list from highlights

If you have highlighted items in a list, using the Find or Show properties controls, this control will remove all but the highlighted items from the list.

Filter > Make list from non-highlighted items

If you have highlighted items in a list in the lower left panel, using the Find or Show properties controls, this control will remove all the highlighted items from the list, leaving the non-highlighted items only.

Options > Remove unassigned characters from list

If you have unassigned characters in a list in the lower left panel this control will remove them from the list.

Options > Show U+ in lists

If this is checked, hex code point numbers in lists in lists will be preceded by U+. The default is just the number.

Options > List format

This allows you to change the order and items in lists appearing in the lower left panel. By default, you would see something like this:

0968 २ DEVANAGARI DIGIT TWO

With this control you can position the character before or after the number (or both!) or remove it altogether. You can also specify whether the list should show the number and/or the name of the character.

This control is provided for people who want some control over how the list will look when copied and pasted into their text.

Options > Hide numbers around matrix

This allows you to hide the column and row numbers around a table. The default is to show the numbers.

Options > Left panel max height (pixels)

This control allows you to change the height of the display box in the lower left panel. Just enter the height you want in pixels (just the number).

This is particularly useful when you are dealing with lists on a small screen such as a netbook. If you set the height to something like 400px, you can scroll through a long list, but still see the details in the right panel when you click on a character in the list.

There is another control just below this with some preset size selections. This will be added to over time.

Options > Left panel font size

This control allows you to increase the size of the characters in the lower left panel (independently of text elsewhere on the page). Note: It has no effect when viewing characters as graphics.

To see all text larger, use the normal browser method for zooming (eg. Ctrl++ and Ctrl+- in browsers on Windows).

You can tailor the program by...

Using URIs to start up UniView with data in left or right panels

This is useful for pointing people to particular information using a URI, for example in email. By providing query parameters in the URI you can start up UniView with specific information displayed as follows:

  • uniview/?char=<hex-codepoint> Displays the information about the specified character in the right panel on startup.
    Eg. http://r12a.github.io/uniview/?char=0449
  • uniview/?block=<block-name> Displays the specified block in the left panel on startup. The block name must be written exactly as it appears in the pulldown selector at the top left of the page, except that any spaces must be replaced by either underscores ('_') or '%20', and case is not important. (Tip: when typing the this in a browser's address bar you can just use spaces.)
    Eg. http://r12a.github.io/uniview/?block=latin%20extended-b
  • uniview/?codepoints=<list-of-hex-codepoints> A list of the characters will be displayed in the left panel. Any spaces must be replaced by either underscores ('_') or '%20', and case is not important.
    Eg. http://r12a.github.io/uniview/?codepoints=449%201234%2065

You should only use one of these query parameters in a single call to UniView.

Change history

Changes in version 8.0.0

This version adds 1,945 new non-ideographic characters encoded in Unicode 8.0.0 (including 6 new scripts).

I also finally fixed the Show Age filter, and brought it up to date.

The github site now holds images for all 28,000+ Unicode codepoints other than Han ideographs and Hangul syllables (in two sizes).

Changes in version 7.0.0

This version updates the app per the changes during beta phase of the specification, so that it now reflects the finalised Unicode 7.0.0.

The initial in-app help information displayed for new users was significantly updated, and the help tab now links directly to the help page.

A more significant improvement was the addition of links to character descriptions (on the right) where such details exist. This finally reintegrates the information that was previously pulled in from a database. Links are only provided where additional data actually exists.

Rather than pull the data into the page, the link opens a new window containing the appropriate information. This has advantages for comparing data, but it was also the best solution I could find without using PHP (which is no longer available on the server I use). It also makes it easier to edit the character notes, so the amount of such detail should grow faster. In fact, some additional pages of notes were added along with this upgrade.

A pop-up window containing resource information used to appear when you used the query to show a block. This no longer happens.

Changes in version 7beta

This version adds the 2,834 new characters encoded in the Unicode 7.0.0 beta, including characters for 23 new scripts. It also simplified the user interface, and eliminated most of the bugs introduced in the quick port to JavaScript that was the previous version.

Some features that were available in version 6.1.0a are still not available, but they are minor.

Significant changes to the UI include the removal of the 'popout' box, and the merging of the search input box with that of the other features listed under Find.

In addition, the buttons that used to appear when you select a Unicode block have changed. Now the block name appears near the top right of the page with a icon. Clicking on the icon takes you to a page listing resources for that block, rather than listing the resources in the lower right part of UniView's interface.

UniView no longer uses a database to display additional notes about characters. Instead, the information is being added to HTML files. When one of these files contains information for a particular block, you'll see a link to it from the detailed information for a particular character. At some future point, I may pull the information for that character into UniView, as before, but for the time being clicking on the Character notes link opens the page in a separate window. Initially such a link is only available for Tibetan, but I will add more from time to time.
